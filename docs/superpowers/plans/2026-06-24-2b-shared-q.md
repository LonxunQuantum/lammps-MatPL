# Two-Body Descriptor Shared-q Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Accumulate radial descriptors across neighbors in per-thread shared memory and write each `g_Fp` value once.

**Architecture:** Extend the existing dynamic shared-memory region with a descriptor-major `s_q` tile. Keep V3-A's register-resident per-neighbor `gn12`, but redirect the final per-neighbor accumulation from global `g_Fp` to `s_q`, followed by one global store per descriptor after the neighbor loop.

**Tech Stack:** CUDA 12.4, C++17, LAMMPS/Kokkos, NSYS, NCU, WMoTaV frozen regression input.

### Task 2: Add the shared-q accumulator

**Files:**
- Modify: `nep_gpu/force/nep_kernal_function.cuh:48-155`
- Modify: `nep_gpu/force/nepkk.cu:812-829`

- [ ] **Step 1: Partition dynamic shared memory**

Keep `s_c` at offset zero and define `s_q` after
`sim_num_types * sim_num_types * n_max_radial_plus1 * basis_size_radial_plus1`
coefficient elements.

- [ ] **Step 2: Initialize per-thread accumulators**

For every `n < n_max_radial_plus1`, set
`s_q[n * blockDim.x + threadIdx.x]` to zero before entering the neighbor loop.
No block synchronization is needed because each thread exclusively owns its
descriptor slots.

- [ ] **Step 3: Redirect per-neighbor accumulation**

Replace `g_Fp[n * nlocal + atomi] += gn12[n]` with
`s_q[n * blockDim.x + threadIdx.x] += gn12[n]`, preserving the current fixed
upper-bound unrolled loop.

- [ ] **Step 4: Store final radial descriptors once**

After the neighbor loop, write
`g_Fp[n * nlocal + atomi] = s_q[n * blockDim.x + threadIdx.x]` for active
radial channels.

- [ ] **Step 5: Extend launch shared memory**

Launch with compact C2 bytes plus
`BLOCK_SIZE32 * n_max_radial_plus1 * sizeof(NEP_FLOAT)`.
